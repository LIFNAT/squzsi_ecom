# Squishy CS (สกุชชี่ CS)

แพลตฟอร์มดิจิทัลสำหรับการจัดการคลังสะสมและร้านค้าออนไลน์ที่ยกระดับประสบการณ์การซื้อสกุชชี่ จาก "ของเล่น" สู่ "เครื่องมือบำบัดทางอารมณ์" (Emotional Tool) ที่ทรงพลัง

## 🎯 วัตถุประสงค์ของโครงงาน
1.  **พัฒนาระบบแพลตฟอร์มดิจิทัล:** เพื่อสร้างระบบเว็บไซต์ที่ทันสมัย ตอบสนองความต้องการใช้งานผ่านเทคโนโลยีที่รวดเร็ว (Next.js, Tailwind CSS)
2.  **ยกระดับประสบการณ์เชิงสัมผัสและอารมณ์:** ออกแบบแพลตฟอร์มให้เป็นเครื่องมือช่วยผ่อนคลาย โดยเน้นการนำเสนอสื่อมัลติมีเดียที่สะท้อนคุณค่าทางจิตใจ
3.  **สร้างระบบนิเวศการสะสมที่ยั่งยืน:** พัฒนาระบบรองรับการซื้อขายสินค้า Limited Edition และตลาดมือสองที่มีความน่าเชื่อถือ
4.  **พัฒนาระบบจัดการหลังบ้านที่มีประสิทธิภาพ:** ระบบวิเคราะห์ข้อมูลเชิงลึก (Analytics) และจัดการคลังสินค้าที่แม่นยำ ปลอดภัยด้วยมาตรฐานระดับสากล

---

## 🛠 เทคโนโลยีที่ใช้ (Tech Stack)
*   **Frontend:** [Next.js](https://nextjs.org/) (SSR for performance & SEO), [Tailwind CSS](https://tailwindcss.com/)
*   **Backend:** [Node.js](https://nodejs.org/) & [Express.js](https://expressjs.com/) (Middleware & Business Logic)
*   **Database & Auth:** [Supabase](https://supabase.com/) (PostgreSQL, Real-time Subscription, Row Level Security)

---

## 🏗 สถาปัตยกรรมระบบ (SDLC)
กระบวนการพัฒนาโครงการนี้ยึดตามหลักมาตรฐาน **SDLC (Software Development Life Cycle)**:
1.  **Planning & Requirement Analysis:** วิเคราะห์ความต้องการเชิงธุรกิจและฟังก์ชันระบบ
2.  **Defining Requirements:** จัดทำเอกสารข้อกำหนดซอฟต์แวร์ (SRS)
3.  **Designing:** ออกแบบ UI/UX และ Database Architecture
4.  **Building / Developing:** พัฒนาระบบด้วย Next.js และ Express.js
5.  **Testing:** ทดสอบระบบเพื่อความถูกต้องและความปลอดภัย
6.  **Deployment:** ติดตั้งระบบบน Server พร้อมรองรับผู้ใช้งานจริง
7.  **Maintenance:** การดูแลรักษาและปรับปรุงฟีเจอร์อย่างต่อเนื่อง

## 🚀 ขอบเขตของระบบ

### 1. ฝั่งหน้าบ้าน (Customer Facing)
*   **ระบบสินค้า:** ค้นหา กรองข้อมูล และดูรายละเอียดสินค้าเชิงลึก
*   **ระบบตะกร้าสินค้า:** จัดการรายการสินค้าที่เลือกก่อนการสั่งซื้อ
*   **ระบบชำระเงิน:** รองรับช่องทางที่หลากหลายและปลอดภัย
*   **ติดตามสถานะ:** ตรวจสอบสถานะคำสั่งซื้อแบบ Real-time
*   **ระบบโปรโมชั่น:** จัดการรหัสส่วนลดและแคมเปญต่างๆ

### 2. ฝั่งหลังบ้าน (Admin Facing)
*   **จัดการสต๊อก:** เพิ่ม/แก้ไขสินค้า และแจ้งเตือนเมื่อสินค้าใกล้หมด
*   **เดชบอร์ด:** แสดงวิเคราะห์ภาพรวมธุรกิจและพฤติกรรมลูกค้า
*   **จัดการสมาชิก:** ดูแลข้อมูลผู้ใช้และระบบ CRM

## 💡 คุณค่าของโครงการ (Value Proposition)
*   **Sensory & Emotional Relief:** รองรับไลฟ์สไตล์การคลายเครียดของคนรุ่นใหม่
*   **Scalable Architecture:** ออกแบบมาเพื่อรองรับจำนวนผู้ใช้ที่เพิ่มขึ้น
*   **High Security:** ปกป้องข้อมูลลูกค้าด้วยระบบ Row Level Security (RLS) จาก Supabase
